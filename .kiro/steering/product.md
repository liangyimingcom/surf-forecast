# Product Steering — 浪报 Surf Forecast（v2）

## 一句话定位

一个**面向会员的动态冲浪浪报网站**：会员查询指定位置，得到逐日、可执行、讲清"为什么"的冲浪行动报告；并用**昨日回看校验机制**让用户拿自己的真实体感评估预报准度，建立信任。

## v2 与 v1 的根本差异

v1 把产品定义为"分析引擎"（CLI/库）。但项目实际演进的重心是**Web 会员产品 + 信任机制**。v2 据此重写：

| 维度 | v1 | v2（本版） |
|------|----|----|
| 产品主体 | 分析引擎（出 Markdown 报告） | 动态 Web 会员站（引擎是其后端模块） |
| 交互 | 无 | 小白/高手双模式、日期条、SVG 图表、自评 |
| 信任机制 | 仅"数据可信度声明" | **昨日回看校验**：历史预报 vs 用户体感 |
| 风况 | 仅风速 | **离岸/向岸(offshore/onshore)风质**编码 |
| 周期 | 单口径 | Tm/Tp **双口径** |
| 时间 | UTC+8 | 显式 **GMT+8 校准时间戳** |
| spec 数量 | 1 | 3（引擎 / Web / 校验反馈） |

## 我们解决的问题

公开浪报只给参数曲线，不给判断，更不给"凭什么信你"的依据。本产品三层价值：

1. **判断**：多参数博弈评分 → 哪天去、几点到、什么板、冲多久。
2. **教学**：解释物理机制（频散、陆海风/morning glass、离岸风梳面、涌浪生命周期）。
3. **信任**：昨日回看——用用户能亲身验证的过去，为不可验证的未来背书。

## 目标用户

| 人群 | 诉求 | 产品对应 |
|------|------|---------|
| 中高级冲浪者 | 精确窗口、板型、跨日挑最佳 | 高手模式 + 排名 + 图表 |
| 进步期初学者 | 安全边界、友好窗口 | 小白模式 + 一句话建议 |
| 上班族冲浪者 | 哪天值得请假 | Hero 必冲卡 |
| 浪报学习者 | 理解为什么 | 物理小课堂 + 双口径 + 离岸风 |
| 怀疑论者 | 凭什么信这套预报 | **昨日回看校验** |

## 产品原则（不可妥协）

1. **判断导向**，不是数据搬运——每天落到几点/什么板/冲多久。
2. **解释为什么**——强制物理机制叙事。
3. **多参数博弈**——核心信条：**冲浪日的上限由最差的参数决定，不由最好的参数决定。**
4. **数据诚实**——推算/中期预报/口径/时区必须显式标注；严禁用正确物理套不存在的数据（见 reference 事实核查）。
5. **风向是一等参数**——离岸(梳面·最佳)/侧岸/向岸(吹乱)必须在判断与图示中体现。
6. **信任可验证**——历史校验用真实历史回算数据，不可造假。
7. **只评估白天可冲时段**；所有时间 GMT+8。

## 关键成功指标

- 每天都有"综合分+最佳窗口+板型+行动时间表"。
- 排名差异能被参数阈值变化解释。
- 会员视图体验达到或超过 [reference 中的 UI 黄金样本](../../reference/reports/ui-golden-sample-浪报MVP.html)。
- 昨日回看的历史数据与用户体感可对照，自评能转化为系统性偏差校准。
- 所有可独立验证的硬事实（星期、月相/潮汐、水温、时区）零错误。

## 范围边界

**做**：多位置查询、会员鉴权、逐日深度分析+图表、离岸风、双口径、昨日回看校验、缓存。

**暂不做**：支付扣费、社交登录、推送、多语言、多浪点横向对比、长期历史趋势归档。见 [docs/kiro-development-guide.md](../../docs/kiro-development-guide.md) 路线图。
