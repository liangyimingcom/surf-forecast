## 一、问题诊断

会员视图高手模式三处图表**数据标签过载**：15 个白天时段每格叠多重文字标签，塞进 ~600px 宽 → 重叠/裁切/竖排断行。

| 区域 | 拥挤根因 |
|------|---------|
| ① 浪高·涌浪·周期图 | 每时段 4 类数字（Tm/Tp/浪高/涌浪）× 15 = ~60 个标签糊成一片 |
| ② 离岸风质条 | 每格 4 行文字（箭头/风速/时段/向岸中文），15 格 flex:1 挤压致最后一格裁切、「17时」竖排断行 |
| ③ 潮汐图风向点 | 曲线下方 15 个风速数字标签重叠成团；去数字后 15 个圆点(r6.5)仍相接 |

**核心原则**：数据点不减，**文字标签抽稀 + 去重复 + 给间距**。

---

## 二、优化措施（前 → 后）

### ① 浪高·周期图（waveChart）
- **前**：`d.times.forEach` 中每个 i 都画 浪高数字 + Tm 数字 + Tp 数字 + 时间标签。
- **后**：引入抽稀 `LBL=ceil(n/7)`，`showLbl(i)` 仅每 ~7 个显示一次文字；柱/点/折线全部保留。
```diff
- s+=`<text ...>${d.hs[i]}</text>`               // 每格都标
+ if(showLbl(i)) s+=`<text ...>${d.hs[i]}</text>` // 抽稀
```
标签密度 60→~7 组，红/橙数字不再重叠。

### ② 离岸风质条（windQualityStrip + .windq CSS）
- **前**：`.windq{display:flex} .seg{flex:1}`；每格 `${t}时·${m.label}`（含向岸/离岸中文）。15 格等分 → 每格 ~38px，文字换行、末格裁切。
- **后**：
  - 去掉每格中文 label（颜色 + 底部图例已表达风质）
  - CSS 改**横向滚动**：`.windq{overflow-x:auto}`，`.seg{flex:0 0 auto;min-width:42px;white-space:nowrap}`
  - 格子够宽后恢复每格显示时段
```diff
- .windq .seg{flex:1;padding:5px 1px 4px}
+ .windq{overflow-x:auto} .windq .seg{flex:0 0 auto;min-width:42px;white-space:nowrap}
- <span class="wq-h">${t}时·${m.label}</span>   // 07时·向岸
+ <span class="wq-h">${t}时</span>               // 仅时段, 滑动看全天
```
不再挤压/裁切/竖排断行；左右滑动查看全部时段。

### ③ 潮汐图风向点（windTideChart）
- **前**：每点上方标风速数字 `${d.wind[i]}`（与①风况折线+②风质条重复三次）；15 个 r6.5 圆点相接。
- **后**：
  - 删除重复风速数字（风速看②即可）
  - 风向点抽稀 `WPT=ceil(n/8)`（15→~8）+ 半径 6.5→6
```diff
- s+=`<text ...>${d.wind[i]}</text>`   // 删除重复数字
- s+=`<circle ... r="6.5">`
+ if(i%WPT!==0 && i!==n-1) return;     // 抽稀
+ s+=`<circle ... r="6">`
```
曲线下方那团数字消失，圆点不再相接。

## 四、设计原则沉淀（可复用）

1. **标签抽稀优先于删数据**：`step=ceil(n/K)`，K 取 7-8，保证任意宽度下标签数恒定，不随时段数膨胀。
2. **去跨图重复**：同一指标（风速）勿在多图重复标注，只在最合适的一张标。
3. **窄容器用横向滚动而非等分挤压**：固定格宽 + `overflow-x:auto` + `white-space:nowrap`，避免文字换行/裁切。
4. **颜色+图例替代逐格文字**：风质用色块+一次性图例，胜过每格中文标签。

> 未采用方案 B（图表 SVG 横向滚动 + 响应式字号）——方案 A + 本轮续优化已解决全部可见拥挤，成本更低。
