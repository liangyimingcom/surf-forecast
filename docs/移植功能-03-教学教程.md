# 移植功能 · 教学教程（surf-forecast）

> 面向开发者：如何在 surf-forecast 前端新增/扩展这类功能，架构、代码地图、复用模式、红线与测试。

## 一、5 分钟上手
```bash
cd surf-forecast-kiro-v2 && . .venv/bin/activate
# 起后端(内存 store, 前端由后端 ALB/本地直供 /)
SF_FRONTEND=web/浪报MVP.html PYTHONPATH=src uvicorn web.app:app --host 127.0.0.1 --port 8848
# 浏览器开 http://127.0.0.1:8848 （移动端视图最佳）
# 跑 E2E
node web/e2e/new_features.mjs http://127.0.0.1:8848
# 截图
node web/e2e/shots.mjs http://127.0.0.1:8848
```

## 二、架构与边界
```
web/浪报MVP.html  单文件前端(HTML+CSS+内联<script>)
  ├─ render() 管理的核心区: #strip/#hero/#cards/#extras/#verify（浪报主体，勿触碰）
  ├─ 附加功能区(本次移植): #spotFav(收藏/搜索/地图) #annc #feedback #about
  │                        #newswall #carpool #volume #nearby #livecams
  └─ 数据: DAYS/HISTORY(来自 /api/report) · SAVED_SPOTS(来自 /api/spots) · 示例常量
src/web/     FastAPI 后端(app.py 路由 / deps.py get_report / db.py 存储)  ← 本次零改动
src/surf_forecast/  引擎(physics/scoring/validate/render)                 ← 本次零改动
```
**关键原则**：新功能一律**附加式**——独立 section + 独立 JS 函数 + `load` 事件初始化，**不修改 `render()`** 与后端/引擎。

## 四、如何新增一个类似功能（模式）
1. **加 section**：在 `#annc` 之后插入 `<section id="xxx">`（不进 `render()` 区）。
2. **加 CSS**：在 `</style>` 前追加 `.xxx-*` 样式。
3. **加数据**：真实数据走 `/api/*`（复用 `fetch(..., {credentials:'include'})`）；无上游则用**前端示例常量**并标「示例数据」。
4. **加渲染函数** + `window.addEventListener('load', initXxx)`。
5. **转义**：动态字段一律经 `_favEscape()` 防注入。
6. **验证**：`node --check`（提取 `<script>`）→ 扩 `web/e2e/new_features.mjs` 断言 → 起后端跑 E2E。

## 五、逆向/移植踩坑（本次真实经历）
1. **并发写冲突**：两个 Goal 驱动器同改文件导致 tasks.md 记录与实际 HTML 失同步 → 教训：**单一驱动器**，改前先 `grep` 确认函数真存在，勾选必须与文件一致。
2. **Leaflet 隐藏容器尺寸错乱**：地图容器由 `display:none` 转显示需 `invalidateSize()` 重算。
3. **收藏面板依赖已保存浪点**：`SAVED_SPOTS` 空则隐藏——E2E 需先经 API 建浪点再断言。
4. **createFeedback 需登录**（石老人上游）→ surf-forecast 无对应上游，改为**前端示例校验 + 占位**，不写库。

## 六、合规边界（红线）
- 仅公开只读 / 前端示例；不做登录态、支付、社区发布/编辑/二手。
- 不引外部第三方内容 API；联系方式脱敏；直播为演示占位不误导。
- 全程 GMT+8；不改引擎内核；`/api/spots` 全 401；DATA CONTRACT 含 wdeg 不受影响；DynamoDB 写入必过 `float→Decimal`（本次无新写入）；pytest 118 零倒退。
